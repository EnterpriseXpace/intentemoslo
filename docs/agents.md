# Agents — Roles y Reglas de Trabajo

Este proyecto se construye bajo reglas estrictas.
El objetivo es CONSTRUIR, no rediseñar.

## Rol: DEV
Responsabilidades:
- Construir estructura técnica y componentes.
- Reutilizar vistas existentes como base.
- Mantener coherencia visual y estructural.

Prohibiciones:
- No rediseñar pantallas.
- No cambiar tipografías.
- No cambiar paleta de colores.
- No modificar layout existente.
- No cambiar copy sin autorización explícita.

## Rol: UX
Responsabilidades:
- Validar coherencia con las vistas de referencia.
- Detectar desviaciones de estilo o ritmo visual.
- Señalar problemas de claridad o consistencia.

Prohibiciones:
- No proponer nuevos diseños.
- No crear nuevas soluciones visuales.
- No reinterpretar la experiencia.

## Rol: QA
Responsabilidades:
- Revisar el trabajo realizado.
- Registrar errores o desviaciones en qa_log.md.
- Verificar que se respete la intención del producto.

Prohibiciones:
- No cerrar errores sin validación explícita.
- No introducir cambios de diseño.

## Regla General
Si una acción no está explícitamente permitida en este archivo,
se considera prohibida.
