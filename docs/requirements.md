# Requirements — Requisitos del Proyecto

Este documento define lo que el proyecto DEBE tener.
No describe diseño ni implementación técnica.

## Requisitos de Producto
1. El producto debe mantener coherencia total con las vistas de referencia ubicadas en /reference_views.
2. El tono del producto debe ser emocional, sobrio y respetuoso.
3. El producto no debe sentirse como terapia, autoayuda ni coaching.
4. El producto no debe empujar decisiones ni dar órdenes al usuario.

## Requisitos de Diseño
5. Tipografías y paleta de color deben mantenerse coherentes con las vistas existentes.
6. No se deben introducir nuevos estilos visuales.
7. No se deben introducir nuevos componentes gráficos sin autorización.

## Requisitos de Construcción
8. El desarrollo debe priorizar estructura y mantenibilidad.
9. El copy puede considerarse provisional y ajustarse más adelante.
10. No se debe rediseñar mientras se construye.

## Requisito Fundamental
11. Si un cambio rompe la coherencia visual o emocional del producto,
   el cambio se considera inválido.   

## Fuente Visual Obligatoria
Las vistas ubicadas en la carpeta /reference_views
son la referencia visual y estructural oficial del proyecto.

Cualquier construcción debe alinearse a estas vistas.
No deben reinterpretarse ni rediseñarse.
